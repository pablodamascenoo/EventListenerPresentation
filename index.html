O que são Event Listeners?

  • Event Listeners são procedimentos ou funções em um programa que esperam um certo evento acontecer.
  • Os event listeners trabalhados nessa apresentação serão divididos em duas categorias principais: mouse events e keyboard events.
  • Para adicionar um event listener, é preciso utilizar o método addEventListener para o elemento que vai ser escutado.
  • O addEventListener possui 3 parâmetros: o tipo de evento, a função a ser executada e um boolean que indica se o evento será executado na fase de capture ou de bubbling

O que são Capture e bubbling phases?

Ambos se tratam sobre como os eventos de mesmo tipo irão se comportar em um sistema hierarquico.

Capture

A fase de captura ocorre quando o terceiro argumento do addEventListener é dado como true, nela, os eventos ocorrem do elemento mais externo para o mais interno. por exemplo, esse quadrado rosa abaixo é pai do quadrado branco e ambos possuem o addEventListener com o modo capture.

Bubbling

A fase de borbulha ocorre quando o terceiro argumento do addEventListener é dado como false que vem dessa forma como padrão. Mouse Events

Eventos de mouse são eventos acionados assim que o usuário interage na página com o seu mouse. Os principais eventos utilizados são: click, dbclick, mouseup e mousedown.

Mouse Events vs onclick

Embora o onclick e o event listener possuam a mesma finalidade, existem diferenças cruciais entre eles.

Mouse events

  • só pode ser adicionado por script
  • pode acionar vários eventos por vez
  • versões antigas de alguns browsers usam attachEvent ao invés de eventListener

Onclick

  • Pode ser adicionado como atributo no HTML
  • só pode acionar um único evento
  • funciona em todos os browsers
